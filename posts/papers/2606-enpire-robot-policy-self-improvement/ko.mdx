## TL;DR

ENPIRE는 coding agent가 실제 로봇 실험을 직접 돌리고, 실패 로그를 읽고, 정책 학습 코드를 수정하고, 다음 실험을 다시 배치하는 **physical autoresearch** harness다. 핵심은 로봇 정책 개선을 "reset -> execute -> verify -> refine"이라는 반복 가능한 최적화 루프로 바꾼다는 점이다.

## 문제

Dexterous manipulation은 아직도 사람의 supervision과 algorithmic engineering에 크게 의존한다. 데이터 수집, 실패 후 reset, 성공 여부 검증, 학습 코드 수정이 모두 사람 손을 타기 때문에 실제 로봇 정책을 빠르게 개선하기 어렵다. 최근 coding agent는 디지털 환경에서 실험 자동화를 잘하지만, 물리 세계에서는 로봇을 안전하게 reset하고, 실제 실행 결과를 검증하고, 여러 로봇에 실험을 분산하는 인터페이스가 부족하다. ENPIRE가 다루는 문제는 바로 이 간극이다.

## 핵심 아이디어

ENPIRE는 로봇 정책 개선을 coding agent가 다룰 수 있는 네 모듈로 분해한다.

- **Environment (EN)**: 사람의 피드백을 받아 자동 reset, safety boundary, reward/verification API를 구성한다.
- **Policy Improvement (PI)**: heuristic learning, behavior cloning, offline/online RL, tool calling 같은 정책 개선 루틴을 실행한다.
- **Rollout (R)**: 한 대 또는 여러 대의 실제 로봇에서 policy를 평가하고 로그를 축적한다.
- **Evolution (E)**: agent가 로그와 문헌을 분석하고, 학습 인프라와 알고리즘 코드를 수정하며, 실패 모드를 줄이는 다음 실험을 만든다.

<Collapsible title="구체적 방법">

ENPIRE의 중요한 설계는 environment construction과 policy improvement를 분리한 것이다. 첫 단계에서는 coding agent가 사람의 feedback을 받아 task-specific environment API를 만든다. 이 API에는 reset, observation, step, reward, success verification이 들어가며, 한 번 안정화되면 이후 정책 개선 단계에서는 immutable interface로 사용된다.

두 번째 단계에서는 agent가 이 environment API만 보고 정책 개선을 반복한다. agent는 behavior cloning의 regularization weight를 바꾸거나, online RL과 demo data mixing을 조정하거나, perception tool call과 motion planning을 결합하는 식으로 실험 branch를 만든다. 여러 로봇이 있을 때는 한 station당 하나의 coding agent를 붙여 hypothesis를 병렬로 테스트하고, Git/log summary를 통해 좋은 branch를 이어받는다.

이 구조는 로봇 실험을 완전히 추상화하지 않는다. 오히려 실제 로봇의 비결정성, contact friction, perception failure, reset variability를 그대로 feedback으로 받아들이고, agent가 그 feedback 위에서 policy recipe를 개선하게 만든다.

## 주요 결과

- Frontier coding agent가 ENPIRE 위에서 Push-T, pin insertion, GPU insertion, zip-tie cutting 같은 실제 dexterous manipulation 태스크를 다루며, 논문 초록 기준 최대 **99% success rate**까지 정책을 개선한다.
- Pin insertion에서는 coding agent가 BC, iterative BC, online/offline/offline-to-online RL, BC regularization 등을 탐색하고, 50연속 성공을 목표로 policy를 개선한다.
- 로봇 플릿 확장은 wall-clock time을 줄인다. Push-T에서는 1대에서 8대로 늘릴 때 normalized score 1.0 도달 시간이 약 5시간에서 2시간으로 줄고, pin insertion에서는 near-perfect success rate 도달 시간이 1.5시간 이상에서 약 40분으로 줄어든다.
- 비용은 공짜가 아니다. 1대에서 8대로 늘리면 GPU utilization과 token consumption은 올라가지만 per-robot utilization은 낮아진다.

## 달성점과 한계점

새롭게 달성한 점:

- Coding agent 기반 autoresearch를 디지털 ML 실험이 아니라 실제 dexterous robot policy improvement로 옮겼다.
- 자동 reset과 자동 verification을 갖춘 environment API를 먼저 만들고, 그 위에서 policy learning을 자동화하는 두 단계 구조를 제안했다.
- 한 대의 로봇이 아니라 여러 robot station으로 hypothesis search를 병렬화하고, MRU와 MTU로 physical autoresearch의 자원 효율을 측정했다.

한계점:

- 저자들이 직접 지적하듯 robot resource와 compute resource가 완전히 포화되지 않는다. agent가 로그를 읽고 코드를 쓰고 LLM 응답을 기다리는 동안 로봇은 놀 수 있다.
- 플릿을 키우면 wall-clock time은 줄지만 token cost가 비선형적으로 증가한다. 즉 더 빠른 연구 속도를 토큰 비용과 coordination overhead로 산다.
- 🤖 실험 환경은 매우 강하게 수직 통합된 로봇 플릿과 내부 tool API를 전제한다. 다른 연구실이 같은 수준의 자동 reset, verification, data logging, agent sandbox를 재현하기는 쉽지 않다.
