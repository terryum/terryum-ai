## TL;DR

LLM system performance is determined not just by model weights but by the harness—the code that stores, retrieves, and presents information to the model. Meta-Harness automatically searches over harness code by giving proposer agents full filesystem access to prior candidates' code, traces, and scores, outperforming hand-engineered baselines across three domains.

## Problem

LLM system performance depends critically on the **harness**—the code that determines what information to store, retrieve, and present to the model. Yet harnesses are still designed largely by hand, and existing text optimizers (OpenEvolve, etc.) compress feedback too aggressively, losing the rich context of "why a strategy failed." This makes them poorly matched to the harness optimization setting.

## Key Idea

Meta-Harness is an **outer-loop system** that searches over harness code. The key differentiator from existing text optimizers is giving the proposer agent **full access to all prior candidates' source code, execution traces, and scores** through a filesystem:

- **Filesystem-based experience accumulation**: Each iteration's harness code, reasoning traces, and evaluation scores are stored in a directory structure
- **Agentic proposer**: Reads the entire filesystem, analyzes success/failure patterns, and proposes new harness code
- **Harness = LLM + Code**: The harness encompasses all information storage, retrieval, and presentation logic—a much broader search space than prompt optimization alone

The core insight: **preserving feedback without compression** allows the agent to reason about "why this strategy failed" and make structural changes.

### Search Loop Structure

Each iteration:
1. **Proposer agent** explores the filesystem — reading prior candidates' code, traces, scores
2. Writes new harness code in Python
3. Evaluates the harness on evaluation tasks
4. Stores results (code + traces + scores) in the filesystem

### Proposer's Information Access

The critical difference from existing text optimizers:
- **OpenEvolve**: Provides only scores and short feedback strings
- **TTT-Discover**: Provides only recent K candidates' code
- **Meta-Harness**: Provides all prior candidates' **full source code + execution traces + scores** via filesystem

Table 3's ablation shows that source code + scores alone outperform baselines, but adding **execution traces** further improves performance.

### Application Domains

1. **Online text classification**: Harness stores/retrieves prior examples for few-shot context construction
2. **Retrieval-augmented math reasoning**: Harness retrieves relevant problems from a 535K corpus
3. **Agentic coding**: Harness determines agent tool usage, execution strategy, and error handling

## Key Results

- **Text classification**: **+7.7 points** over SOTA context management (ACE) with **4x fewer context tokens**. Catches up to OpenEvolve and TTT-Discover in just 4 evaluations, ending **10+ points above** all baselines
- **IMO-level math**: A single discovered harness improves accuracy by **+4.7 points** on average across 5 held-out models over no-retrieval baseline, outperforming strongest fixed baseline (BM25 Retrieval)
- **TerminalBench-2**: **47.7% pass rate** on Claude Opus 4.6, surpassing hand-engineered Terminus-KIRA (44.7%). **37.6%** on Haiku 4.5, ranking #1 among all Haiku agents

## Achievements and Limitations

### What's New
- **Harness auto-optimization paradigm**: Goes beyond prompt optimization to automatically search over the full information storage, retrieval, and presentation logic
- **Value of rich feedback**: Demonstrates that preserving uncompressed feedback (source code + traces) enables structural improvements
- **Cross-model transfer**: A single discovered math harness consistently improves 5 different held-out models

### Limitations
- **Author-noted**: High search cost (multiple LLM calls + evaluation per iteration) may make manual design more efficient for small-scale tasks. Currently validated on only 3 domains
- 🤖 Filesystem-based experience accumulation causes the proposer's context to grow with iterations, potentially hitting context window limits, and no scaling strategy is provided for this
