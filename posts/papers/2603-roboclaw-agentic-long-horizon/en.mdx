## TL;DR

A framework that unifies VLA data collection, policy learning, and deployment under a single VLM agent, using forward-inverse action pairs (EAP) for autonomous self-resetting data collection that minimizes human intervention while significantly improving long-horizon manipulation performance.

## Problem

VLA systems show strong potential for language-driven robotic manipulation, but scaling to long-horizon tasks remains challenging. Existing pipelines separate data collection, policy learning, and deployment, requiring manual environment resets after each trial and resulting in brittle multi-policy execution. The semantic mismatch between collection and deployment environments further degrades multi-policy robustness.

## Key Idea

RoboClaw uses a VLM as a meta-controller to unify data collection, policy learning, and task execution under a **single agent**:

- **Entangled Action Pairs (EAP)**: Couples forward manipulation behaviors (e.g., placing object in drawer) with inverse recovery actions (e.g., removing from drawer) to form **self-resetting loops** for autonomous data collection without human resets
- **MCP Tool Interface**: Exposes policy start/stop, environment observation, robot stats, and human-call tools through the Model Context Protocol for standardized agent-robot interaction
- **Structured Memory**: Three-layer memory ($r_t$ role identity, $d_t$ task-level memory, $w_t$ working memory) maintains decision context across the robot lifecycle
- **Unified Deployment**: The same agent performs high-level reasoning and dynamic policy orchestration during both collection and deployment, eliminating semantic mismatch

<Collapsible title="Detailed Method">

### Entangled Action Pairs (EAP)

The core of EAP is simultaneously learning a **forward policy** and an **inverse policy** for each manipulation task:

1. **Initial human demonstrations**: Collect a small set of forward + inverse demonstrations (warmup)
2. **Autonomous rollout**: Agent executes forward policy → evaluates success/failure → executes inverse policy to reset environment → repeat
3. **Streaming updates**: Newly collected data streams into the VLA Policy Pool for progressive policy refinement
4. **Recovery failure**: When autonomous recovery is unsuccessful, the agent invokes the `Call Human` MCP tool for human intervention

### VLM Agent Decision Making

At each step, the agent performs 5-step CoT reasoning:
1. What have I observed in the scene?
2. What is my current objective or subtask?
3. What are the success criteria for this subtask?
4. Does the current state satisfy the success criteria, or am I stuck/failing?
5. Based on this evaluation, what should I do?

### Policy Learning

Fine-tunes π₀.₅ via LoRA (rank=16, alpha=16, 108 training steps, warmup 100 steps, lr=$2.5 \times 10^{-4}$) to create task-specific policies. Each policy is independently trained and checkpointed; during deployment, the agent dynamically activates and switches between policies.

## Key Results

- **Data collection efficiency**: **53.7% reduction in human time investment** compared to manual collection for the same data volume. Human intervention during rollout reduced by **~8x**
- **Policy improvement**: Over 5 iterations, forward policy success rates consistently improve (Body Lotion 21/50 → 43/50, Primer 21/50 → 40/50)
- **Long-horizon tasks**: On vanity table organization (4 sequential subtasks), **25% success rate improvement** over baselines, outperforming both end-to-end VLA and open-loop multi-policy pipelines

## Achievements and Limitations

### What's New
- **Unified robot lifecycle**: Integrates data collection, learning, and deployment under a single VLM agent, eliminating semantic mismatch from pipeline separation
- **Autonomous data collection**: EAP-based self-resetting enables continuous on-policy data collection with minimal human intervention
- **Real-world long-horizon validation**: Successfully demonstrated sequential execution of 4 subtasks on an Agibot G1 dual-arm mobile robot

### Limitations
- **Author-noted**: When inverse policies fail, the system falls back to human intervention. EAP applicability to complex irreversible tasks (e.g., pouring liquid) remains untested
- 🤖 Experiments are limited to a single platform (Agibot G1) performing tabletop organization tasks; generalization to diverse robot morphologies or manipulation domains (e.g., deformable objects, tool use) is unvalidated
