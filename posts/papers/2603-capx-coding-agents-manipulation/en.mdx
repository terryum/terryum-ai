## TL;DR

The first systematic framework studying Code-as-Policy agents for robot manipulation, showing that training-free agentic test-time computation (multi-turn interaction, ensemble reasoning) recovers human-level performance, and RL post-training achieves sim-to-real transfer.

## Problem

While VLA models learn manipulation skills from large-scale robot data, Code-as-Policy offers an alternative where LLMs generate code to directly control robots. However, systematic study of how effective this approach is for real manipulation has been lacking. Existing work relies on high-level, human-crafted APIs, and performance degrades sharply when these abstractions are removed. This paper systematically addresses whether coding agents can reach down to low-level robot control.

## Key Idea

CaP-X is a unified framework with three core components:

- **CaP-Gym**: An interactive environment integrating RoboSuite, LIBERO-PRO, and BEHAVIOR, where agents control robots by synthesizing and executing code
- **CaP-Bench**: A benchmark evaluating 12 models across axes of abstraction level (high → low-level API), interaction mode (single → multi-turn), and perceptual grounding (text → vision)
- **CaP-Agent0**: A training-free agent combining visual differencing, auto-synthesized skill library, and multi-model ensemble reasoning
- **CaP-RL**: RL post-training with verifiable rewards for code generation models

The key finding is that **scaling agentic test-time computation**—multi-turn interaction, structured execution feedback, visual differencing, automatic skill synthesis, and ensembled reasoning—substantially improves robustness even when agents operate over low-level primitives.

### CaP-Bench Evaluation Tiers

CaP-Bench is organized along two axes:

**Single-turn evaluation (S1-S4)**: Measures performance degradation as abstraction decreases
- S1: High-level primitives + full state observation
- S2: High-level primitives + vision observation
- S3: Low-level primitives + vision observation
- S4: Low-level primitives + vision observation + no in-context API examples

**Multi-turn evaluation (M1-M4)**: Measures the impact of interaction modes
- M1: Textual feedback
- M2: Direct multimodal visual input
- M3: Visual differencing (visual changes converted to text)
- M4: M3 + low-level API

### CaP-Agent0 Architecture

CaP-Agent0 combines three key mechanisms:

1. **Visual Differencing Model (VDM)**: Converts initial scene description and per-turn changes to text
2. **Auto-synthesized Skill Library**: Automatically accumulates utility code generated during CaP-Bench execution
3. **Parallel Query + Ensemble**: Multiple LLMs (Gemini-3-Pro, GPT-5.2, Claude Opus) independently generate code candidates, and an ensemble agent synthesizes the final code

### CaP-RL

Applies Group Relative Policy Optimization with verifiable rewards to Qwen-2.5-Coder-7B-Instruct:
- Trains on state-based API (S1 tier) to avoid noisy rewards
- 60 iterations per task improve compilation rate and success rate
- Learned policy transfers sim-to-real without abstraction dependence

## Key Results

- **CaP-Agent0**: Achieves success rates **comparable to or exceeding human expert code on 4 out of 7** CaP-Bench tasks, with zero training data
- **VLA comparison**: On LIBERO-PRO, CaP-Agent0 matches OpenVLA, $\pi_0$, $\pi_{0.5}$ performance, showing stronger robustness to instruction perturbations
- **CaP-RL**: In simulation, Cube Lift improves from 28% → 88%, Cube Stack from 4% → 44%. On real Franka Emika robot, Cube Stack improves from Pre-RL 3/25 → Post-RL 19/25, **minimizing the sim-to-real gap**

## Achievements and Limitations

### What's New
- **First systematic Code-as-Policy benchmark**: Large-scale evaluation of 12 models across abstraction and interaction axes
- **Training-free human-level manipulation**: Agentic test-time computation scaling achieves VLA-level performance without training data
- **Code agent sim-to-real transfer**: RL post-trained code generation policy transfers to real robots with minimal gap

### Limitations
- **Author-noted**: Evaluation focuses on single-arm manipulation; bimanual and mobile manipulation remain untested. RL training uses privileged state (S1), and RL under vision observation is unresolved
- 🤖 CaP-Agent0's multi-model ensemble (Gemini-3-Pro + GPT-5.2 + Claude Opus) incurs very high inference costs, making economic viability uncertain for real-time control applications
