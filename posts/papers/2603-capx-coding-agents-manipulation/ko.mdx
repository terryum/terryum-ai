## TL;DR

Code-as-Policy 에이전트가 로봇 조작에서 어디까지 가능한지를 체계적으로 연구한 첫 번째 프레임워크로, 학습 데이터 없이 multi-turn 상호작용과 앙상블 추론만으로 human-level 성능을 회복하고, 강화학습으로 sim-to-real 전이까지 달성했다.

## 문제

VLA(Vision-Language-Action) 모델들이 대규모 로봇 데이터로 학습하여 조작 능력을 갖추는 반면, Code-as-Policy 접근법은 LLM이 생성한 코드로 로봇을 직접 제어하는 대안적 방법이다. 그러나 이 방법이 실제 조작에서 얼마나 효과적인지에 대한 체계적 연구가 부재했다. 기존 연구들은 높은 수준의 추상화된 API에 의존하는데, 이러한 API가 제거되면 성능이 급격히 하락한다. 이 논문은 "코드 에이전트가 low-level 제어까지 도달할 수 있는가?"라는 질문에 체계적으로 답한다.

## 핵심 아이디어

CaP-X는 세 가지 핵심 구성 요소로 이루어진 통합 프레임워크다:

- **CaP-Gym**: RoboSuite, LIBERO-PRO, BEHAVIOR를 통합한 인터랙티브 환경으로, 에이전트가 코드를 합성·실행하여 로봇을 제어
- **CaP-Bench**: 12개 모델을 추상화 수준(high-level → low-level API), 상호작용 방식(single-turn → multi-turn), 지각 기반(텍스트 → 비전)의 다양한 축에서 평가하는 벤치마크
- **CaP-Agent0**: 학습 없이 visual differencing, auto-synthesized skill library, 다중 모델 앙상블 추론을 결합한 에이전트
- **CaP-RL**: verifiable reward 기반 강화학습으로 코드 생성 모델을 후학습

핵심 발견은 **agentic test-time computation의 scaling**이다: multi-turn 상호작용, 구조화된 실행 피드백, visual differencing, 자동 스킬 합성, 앙상블 추론을 통해 추상화 없이도 성능을 회복할 수 있다.

### CaP-Bench 평가 체계

CaP-Bench는 두 축으로 구성된다:

**Single-turn 평가 (S1-S4)**: 추상화 수준에 따른 성능 변화를 측정
- S1: high-level primitive + full state observation
- S2: high-level primitive + vision observation
- S3: low-level primitive + vision observation
- S4: low-level primitive + vision observation + no in-context API usage examples

**Multi-turn 평가 (M1-M4)**: 상호작용 방식에 따른 성능 변화
- M1: 텍스트 피드백
- M2: 직접 멀티모달 시각 입력
- M3: visual differencing (시각 차이를 텍스트로 변환)
- M4: M3 + low-level API

### CaP-Agent0 아키텍처

CaP-Agent0는 세 가지 핵심 메커니즘을 결합한다:

1. **Visual Differencing Model (VDM)**: 초기 장면 설명과 각 턴에서의 변화를 텍스트로 변환
2. **Auto-synthesized Skill Library**: CaP-Bench 실행 중 생성된 유틸리티 코드를 자동 축적
3. **Parallel Query + Ensemble**: 여러 LLM(Gemini-3-Pro, GPT-5.2, Claude Opus)이 독립적으로 코드 후보를 생성하고, 앙상블 에이전트가 최종 코드를 합성

### CaP-RL

Qwen-2.5-Coder-7B-Instruct 모델에 verifiable reward 기반 강화학습(Group Relative Policy Optimization)을 적용:
- State-based API(S1 tier)로 학습하여 noisy reward 회피
- 태스크당 60 iteration으로 compilation rate과 성공률 개선
- 학습된 정책이 추상화 의존 없이 sim-to-real 전이

## 주요 결과

- **CaP-Agent0**: 7개 CaP-Bench 태스크 중 **4개에서 human expert 코드와 동등하거나 상회**하는 성공률 달성 (학습 데이터 없이)
- **VLA 비교**: LIBERO-PRO에서 CaP-Agent0가 OpenVLA, $\pi_0$, $\pi_{0.5}$와 비교 가능한 성능을 보이며, 특히 instruction perturbation에서 더 강건한 모습
- **CaP-RL**: 시뮬레이션에서 Cube Lift 28% → 88%, Cube Stack 4% → 44%로 대폭 개선. 실제 로봇(Franka Emika)에서도 Cube Stack Pre-RL 3/25 → Post-RL 19/25로 **sim-to-real gap 최소화**

## 달성점과 한계점

### 새롭게 달성한 점
- **Code-as-Policy의 체계적 벤치마킹**: 12개 모델 × 다양한 추상화/상호작용 축에서의 최초 대규모 평가
- **학습 없는 human-level 조작**: agentic test-time computation scaling으로 VLA 수준 성능을 학습 데이터 없이 달성
- **코드 에이전트의 sim-to-real 전이**: RL 후학습으로 시뮬레이션에서 학습한 코드 생성 정책이 최소 gap으로 실제 로봇에 전이

### 한계점
- **저자 언급**: single-arm 조작 위주 평가로 bimanual이나 mobile manipulation은 미검증. RL 학습이 privileged state(S1) 기반으로 vision observation 하에서의 RL은 미해결
- 🤖 CaP-Agent0의 multi-model 앙상블(Gemini-3-Pro + GPT-5.2 + Claude Opus)은 추론 비용이 매우 높아 실시간 제어가 필요한 실제 응용에서의 경제성은 불확실하다

## Terry's memo

- CaP는 "LLM이 코드를 생성하여 API를 호출하는" 구조인데, 이는 "LLM이 태스크를 분해하여 API를 호출하는" 기존 LLM 플래너(SayCan, RoboClaw 등)와 본질적으로 같다. 코드 출력이냐 자연어 계획이냐는 구현 디테일이지 패러다임의 차이가 아니다.
- CaP의 "제로샷" 장점은 결국 low-level API가 이미 해결해준 범위 안에서만 유효하다. 진짜 어려운 물리적 조작(접촉, 힘 제어 등)은 여전히 텔레옵 데이터 + VLA 학습이 필요하며, 제로샷으로 할 수 있는 것은 고전적 모션 플래닝으로도 가능한 것들이다.
- CaP-X에서 VLA(π₀, OpenVLA)와 비교한 것은 불공정하다. 공정한 비교 대상은 LLM+VLA(RoboClaw 같은 구조)여야 하며, 이 비교는 이 논문에 없다.
- 이 논문의 실질적 기여는 새로운 패러다임이 아니라 **벤치마크**: 12개 모델 × 추상화 수준 × 상호작용 모드의 격자 형태 체계적 평가 데이터를 제공한 것. "LLM 기반 로봇 제어가 어디서 되고 어디서 안 되는지"를 처음으로 정량적으로 측정했다.
- 로봇 제어 스택에서 LLM은 "계획·추론" 층, VLA는 "실행" 층이고, 이 분리는 CaP든 LLM+VLA든 동일하다. 진짜 병목은 LLM의 추론 능력이 아니라, low-level API와 하드웨어의 물리적 조작 능력에 있다.
