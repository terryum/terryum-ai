## TL;DR

OmniRetarget은 인간 전신 시연을 휴머노이드 로봇의 **scene/object interaction까지 보존한 reference motion**으로 바꾸는 리타겟팅 엔진이다. 기존 방식이 keypoint matching만 하다가 penetration, foot skating, contact loss를 만들었다면, 이 논문은 body-object-terrain을 하나의 interaction mesh로 묶고 hard constraint를 걸어 RL이 바로 따라갈 수 있는 데이터를 만든다.

## 문제

휴머노이드가 사람처럼 물체를 들고, 지형을 오르고, 몸 전체를 써서 움직이려면 좋은 reference motion이 필요하다. 하지만 인간 motion을 로봇에 단순히 retarget하면 몸 비율과 관절 구조가 달라서 물체를 뚫고 지나가거나, 발이 미끄러지거나, 손-물체 접촉이 사라진다. 이런 artifact가 있는 reference는 downstream RL에서 더 많은 reward engineering과 post-processing을 요구한다. 결국 문제는 “RL을 더 세게 돌리자”가 아니라, **처음부터 interaction을 잃지 않는 humanoid data generation**을 만드는 것이다.

## 핵심 아이디어

핵심은 사람, 로봇, 물체, 지형을 따로 보지 않고 하나의 **interaction mesh**로 묶는 것이다. Mesh의 vertex는 인간/로봇의 주요 관절점과 물체/환경 표면에서 샘플링한 점들로 구성된다. 그런 다음 source human mesh와 target robot mesh 사이의 Laplacian deformation을 최소화하면서, 로봇이 실제로 만족해야 하는 constraints를 함께 건다.

- **Interaction preservation**: 손-물체, 발-지형, 몸-환경의 상대적 공간 관계를 mesh deformation으로 보존한다.
- **Hard constraints**: collision avoidance, joint limit, velocity limit, stance foot 고정을 optimization 안에 직접 넣는다.
- **Systematic augmentation**: 하나의 시연에서 object pose, object size, terrain height, robot embodiment를 바꾼 새 trajectory를 다시 최적화한다.
- **Minimal downstream RL**: reference가 깨끗하므로 policy는 proprioception 기반으로 5개 reward term과 간단한 domain randomization만 사용한다.

Interaction mesh는 Delaunay tetrahedralization으로 만든다. Body keypoint만 넣는 것이 아니라, object와 environment surface point를 더 조밀하게 샘플링해서 contact 관계가 optimization objective에 실제로 들어오게 한다. 목적함수는 source와 target mesh의 Laplacian coordinate 차이를 줄이는 형태다.

각 timestep에서 로봇 configuration $q_t$를 찾되, 단순 least-squares가 아니라 다음 제약을 함께 만족시킨다.

- signed distance 기반 collision constraint
- joint range와 joint velocity limit
- stance foot position 유지
- temporal smoothness

최적화는 SQP-style solver로 frame-by-frame 풀고, 이전 frame의 해를 warm start로 사용한다. 이 구조의 장점은 robot embodiment가 바뀌어도 keypoint correspondence와 collision model만 바꾸면 Unitree G1, H1, Booster T1 같은 다른 휴머노이드에 같은 원리를 적용할 수 있다는 점이다.

Augmentation은 단순 rigid transform으로 끝나지 않도록 설계되어 있다. 예를 들어 box pickup에서는 object initial pose를 바꾸되 lower body는 nominal trajectory에 anchor하고, upper body가 새로운 coordination을 찾도록 추가 cost를 넣는다. Terrain task에서는 platform height/depth를 바꾸고, elevated terrain에서도 안정적인 contact가 나오도록 ground surface point를 interaction mesh에 샘플링한다.

## 주요 결과

- 데이터 규모: OMOMO box carrying 2.78시간, in-house MoCap 1시간, LAFAN1 4.6시간 등 총 **8시간 이상**의 retargeted trajectory를 만든다.
- Hardware behavior: Unitree G1이 box carrying, platform climbing, slope crawling, wall-flip을 포함한 다양한 whole-body scene interaction을 수행한다.
- 장기 sequence: 4.6kg chair를 들고 플랫폼으로 가져가 stepstone처럼 사용한 뒤 뛰어내리고 roll로 착지 충격을 흡수하는 **30초짜리 multi-stage task**를 보여준다.
- Minimal RL: 정책은 reference motion, proprioception, previous action만 관측하고, body/object tracking, action rate, soft joint limit, self-collision의 **5개 reward term**만 쓴다.
- Augmentation 효과: full augmented dataset으로 train/eval하면 **79.1% success rate**, nominal motion only 평가의 **82.2%**와 비슷해 coverage를 넓히면서 성능 저하를 크게 만들지 않는다.
- Table II 기준 downstream RL success는 robot-object interaction에서 **82.20%**, robot-terrain interaction에서 **94.73%**로 baseline보다 높다.

## 달성점과 한계점

새롭게 달성한 점:

- 휴머노이드 retargeting에서 몸 자세만이 아니라 **object/terrain contact geometry**를 보존해야 한다는 점을 명확히 시스템화했다.
- 하나의 인간 시연을 object pose, object size, terrain height, embodiment variation으로 확장하는 data augmentation path를 제시했다.
- 좋은 reference를 만들면 humanoid RL을 복잡한 reward engineering으로 때우지 않아도 된다는 것을 hardware 결과로 보여준다.
- Code, dataset, trained policies를 공개한다고 명시해 후속 연구가 같은 retargeting/data bottleneck을 검증할 수 있는 발판을 만든다.

한계점:

- 저자 언급: 현재 frame-by-frame optimization은 효과적이지만, noisy video source처럼 더 거친 입력을 다루려면 whole-trajectory를 함께 최적화하는 방향이 필요할 수 있다.
- 저자 언급: 남은 hard motion 중 일부는 현재의 simple RL formulation으로 한계가 있어 curriculum learning이 필요할 수 있다.
- 🤖 재현성 한계: 결과는 Unitree G1, MoCap/OMOMO/LAFAN1, 시뮬레이션/하드웨어 파이프라인이 모두 맞물려 나온다. 다른 연구실이 동일한 hardware evaluation과 retargeting quality metric을 맞추기는 쉽지 않다.
