## TL;DR

π₀.₇은 “로봇에게 무엇을 시킬지”뿐 아니라 “어떤 방식과 품질로 하게 할지”까지 프롬프트에 넣어 학습한 범용 VLA다. 이 rich context conditioning 덕분에 실패·저품질·autonomous rollout·사람 영상·웹 데이터를 함께 쓰면서도 행동을 평균내지 않고, unseen 환경·embodiment·태스크 조합에서 조종 가능한 일반화를 보여준다.

## 문제

기존 VLA는 규모가 커져도 로봇 영역에서 LLM 같은 compositional generalization을 얻기 어려웠다. 여러 로봇, 여러 작업, 여러 전략의 데이터를 단순히 섞으면 모델이 서로 다른 행동 모드를 평균내어 애매한 정책을 만들기 쉽고, 실패 episode나 suboptimal autonomous data는 오히려 성능을 해칠 수 있다. 최근 π₀, Octo, OpenVLA 같은 generalist policy가 나왔지만, unseen instruction, cross-embodiment transfer, 장기 dexterous task, mixed-quality data 활용을 한 모델 안에서 모두 다루기는 부족했다. 이 논문은 더 많은 데이터를 넣는 것보다, 데이터가 어떤 전략과 품질을 담고 있는지 모델이 구분하고 조종할 수 있게 만드는 문제를 다룬다.

## 핵심 아이디어

핵심은 VLA의 입력 context를 단순한 언어 명령에서 **조종 가능한 멀티모달 프롬프트**로 확장하는 것이다.

- **Subtask instruction**: “clean the kitchen” 같은 전체 지시뿐 아니라 현재 해야 할 중간 목표를 언어로 제공한다.
- **Subgoal image**: 경량 world model이 만든 near-future 이미지로 “성공하면 장면이 어떻게 보여야 하는지”를 직접 조건화한다.
- **Episode metadata**: 속도, 품질 점수, mistake 여부 같은 라벨을 붙여 mixed-quality data의 행동 모드를 분리한다.
- **Control mode**: joint-level과 end-effector action을 모두 학습하고, runtime에 control mode를 선택할 수 있게 한다.

<Figure src="./fig-4.png" caption="실험에 사용한 일부 로봇 예시. π0.7은 양팔 모바일 매니퓰레이터, 정적 양팔 로봇, cross-embodiment 실험용 양팔 UR5e 셋업 등 다양한 로봇에서 평가된다." number={4} />

이렇게 하면 실패나 느린 시연도 단순 노이즈가 아니라 “낮은 품질/느린 전략”이라는 조건부 데이터가 된다. 테스트 때는 metadata를 quality 5, mistake false 같은 원하는 모드로 설정해 같은 모델을 더 좋은 행동 쪽으로 steer할 수 있다.

<Collapsible title="구체적 방법">

π₀.₇은 4B VLM 백본, MEM-style video history encoder, 860M action expert를 결합한 약 5B 파라미터 VLA다. 카메라 관측은 최대 4개 view와 최대 6개 history frame을 받으며, subgoal image는 최대 3개 view까지 입력된다. image/history token은 bidirectional attention으로 처리하고, 뒤따르는 텍스트 token은 causal attention을 사용한다.

행동 생성은 기존 π₀ 계열처럼 flow matching action expert가 담당한다. 모델은 관측과 context $c$가 주어졌을 때 action chunk $a$의 근사 log-likelihood를 최대화하도록 학습된다.

$$
\log p(a \mid o, c)
$$

Subgoal image 학습에서는 실제 미래 frame과 world model이 생성한 future image를 함께 사용해 train-test mismatch를 줄인다. 실제 image는 segment end frame 또는 0-4초 뒤 frame에서 샘플링하고, 생성 image도 대량으로 섞는다. Prompt component별 dropout도 중요하다. subgoal image가 있는 batch는 전체의 25%로 제한하고, subtask instruction은 image goal이 있을 때 30% dropout한다. episode metadata는 전체 dropout 15%, 개별 component dropout 5%를 적용해 runtime에서 일부 context가 없어도 동작하게 한다.

테스트 때는 control mode와 episode metadata를 항상 제공한다. metadata는 보통 높은 품질과 no mistake 모드로 설정하고, subtask instruction과 subgoal image는 high-level policy나 사람 코칭으로 갱신한다. action inference와 subgoal/image generation은 비동기로 돌려 최신 context를 VLA가 사용한다.

## 주요 결과

- **Out-of-the-box dexterity**: 에스프레소 만들기, 세탁물 접기, 박스 조립 같은 고난도 태스크에서 별도 task-specific post-training 없이 π₀.₆/π₀.₆* specialist와 맞먹거나 일부 throughput에서 더 높다.
- **Instruction following**: 4개 unseen kitchen과 2개 unseen bedroom에서 14개 open-ended instruction-following scenario를 평가했고, π₀.₅/π₀.₆보다 전반적으로 높은 성공률을 보인다.
- **Cross-embodiment transfer**: source robot과 target robot의 크기와 팔 개수가 달라도 새로운 embodiment에 맞는 전략을 발견한다. UR5e 셔츠 접기에서는 π₀.₇(GC)이 task progress 85.6%, success 80%를 달성했고, 숙련 teleoperator의 90.9% progress, 80.6% success와 근접했다.
- **Mixed-quality data scaling**: metadata가 있는 π₀.₇은 더 크고 평균 품질이 낮은 데이터셋에서도 성능이 계속 오르지만, metadata가 없는 ablation은 저품질 데이터가 늘수록 성능이 악화될 수 있다.
- **Unseen task coaching**: sweet potato를 air fryer에 넣기 같은 unseen long-horizon task를 사람의 단계별 언어 코칭으로 수행하고, 그 coaching episode로 high-level policy를 학습해 추가 teleoperation 없이 자율화할 수 있다.

## 달성점과 한계점

### 새롭게 달성한 점

- VLA에서 **context conditioning을 데이터 스케일링의 핵심 장치**로 다뤘다. 데이터 품질과 전략이 섞여도 metadata를 붙이면 더 많은 데이터를 버리지 않고 활용할 수 있다.
- 단일 모델이 dexterity, instruction following, cross-embodiment transfer, language coaching, mixed-quality data scaling을 모두 보여준다.
- Cross-embodiment transfer가 단순 mimicry가 아니라 target embodiment에 맞는 새로운 전략 발견으로 나타날 수 있음을 실험적으로 보여준다.
- 사람 코칭 episode를 low-level teleoperation 없이 high-level prompting policy로 바꾸는 경로를 제시한다.

### 한계점

- **저자 언급**: seen task는 90%를 넘는 경우가 많지만 unseen task나 unseen task-robot 조합은 60-80% 수준으로 낮아진다. zero-shot generalization은 아직 in-distribution 성능과 차이가 크다.
- **저자 언급**: 대규모 heterogeneous dataset에서는 무엇이 진짜 unseen인지 판정하기 어렵다. 논문의 generalization은 완전히 새로운 skill 생성이라기보다, 넓은 데이터에서 배운 behavior의 remix일 수 있다.
- 🤖 **재현성 한계**: 모델 크기, 데이터 규모, annotation pipeline, 로봇 fleet, evaluation infrastructure가 모두 크고 비공개 요소가 많아 외부 연구자가 결과를 재현하거나 공정 비교하기 어렵다.
