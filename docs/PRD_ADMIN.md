# PRD_Admin.md

## 0) 요약
- 사이트에 관리자 전용 영역(`/admin`)을 추가한다.
- **현재는 관리자 1인 비밀번호 방식**으로 시작하지만, 향후 **비회원/회원/관리자 로그인**으로 확장 가능하게 구조를 잡는다.
- 관리자 접근 시 기존 헤더(Home/Ideas/Research/About) 아래에 **관리자 전용 탭 바 1줄**이 추가로 보인다.
- 초기 관리자 탭: `Stats` (향후 `Discuss`, `Draft` 등 확장)

## 1) 목표
- `/admin` 및 `/admin/*` 접근 시 비밀번호 인증 필요
- 비밀번호는 **코드/깃허브/배포물에 노출되지 않음** (환경변수로만 관리)
- 인증 성공 시:
  - 관리자 탭 바 노출 (`Stats` ...)
  - 관리자 페이지에서만 내부 기능 접근 가능
- 일반 방문자 UX/SEO 영향 최소화 (관리자 UI는 절대 노출하지 않음)

## 2) 비목표 (현재 단계)
- 회원 가입/결제/등급/권한 정책 구현(단, 미래 확장 여지 유지)
- 일반 사용자에게 통계 숫자 공개
- Substack/외부 SSO 연동

## 3) 접근 제어(보안) 설계
- 권장 흐름:
  1) `/admin`에서 비밀번호 입력 폼
  2) 서버에서 검증 후 **서명된 세션 쿠키** 발급
  3) `/admin/*` 요청은 미들웨어로 쿠키 검증
- 비밀번호 저장:
  - 로컬: `.env.local`
  - 배포: Vercel Project Env (Preview/Production)
  - **절대 하드코딩 금지** (예: 현재 값이 “0222”라도 env로만)
- 최소 환경변수:
  - `ADMIN_PASSWORD`
  - `ADMIN_SESSION_SECRET` (쿠키 서명용)

## 4) UI 요구사항
- 관리자만 헤더 아래 “admin 탭 바” 1줄 추가 노출
- admin 탭 바는 작은 높이/절제된 컬러(Teal 과사용 금지), 모바일은 가로 스크롤 허용
- 로그아웃(세션 삭제) 버튼 제공

## 5) 라우팅
- `/admin` : 관리자 홈(초기엔 `/admin/stats`로 이동 가능)
- `/admin/stats` : 통계 대시보드 (PRD_Admin_Stats 참조)
- 미래 확장:
  - `/admin/discuss`
  - `/admin/draft`

## 6) 확장성(미래 로그인 대비)
- 인증 레이어를 “비밀번호”에 고정하지 말고, `isAdminRequest()` 같은 단일 게이트로 감싼다.
- 향후 확장 시:
  - 비회원/회원/관리자 role로 분기
  - 관리자 탭 바는 관리자 role에서만 노출

## 7) 테스트/검증(최소)
- 인증 없으면 `/admin/*` 접근 불가
- 인증 후 새로고침/탭 이동 시 세션 유지
- 로그아웃 동작
- `ADMIN_PASSWORD`가 클라이언트 번들에 포함되지 않음 확인
