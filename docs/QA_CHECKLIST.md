# QA_CHECKLIST.md

## 목적
- v1 개인 홈페이지의 **배포 전 검사**와 **콘텐츠 발행 전 검사**를 표준화한다.
- 원칙: 빠르게 확인 가능해야 하며(짧고 명확), 콘텐츠 손실/링크 오류/언어 오류를 우선 차단한다.

## 사용 규칙
- 이 문서는 체크리스트로만 사용한다 (설명/토론은 다른 문서에 기록).
- `CURRENT_STATUS.md`에는 결과 요약만 남긴다 (append 로그 금지).
- 하나라도 치명 항목 실패 시 배포/푸시 중단.

## A. 배포 전 체크 (사이트/코드 변경 시)
### 1) 빌드/배포
- [ ] 로컬 빌드 성공
- [ ] 타입/린트/테스트(설정된 범위) 통과
- [ ] 프로덕션 환경변수 누락 없음
- [ ] `npm run build:cf` (OpenNext Worker 빌드) 성공
- [ ] Cloudflare DNS/도메인 설정 변경 시 연결 정상 확인

### 2) 라우팅/I18N
- [ ] `/ko`, `/en` 진입 정상
- [ ] 브라우저 언어 기반 첫 진입 라우팅 동작 확인
- [ ] 수동 언어 전환 후 선택값 유지(쿠키/스토리지) 확인
- [ ] 번역 없는 페이지 fallback 동작 확인
- [ ] 메뉴 라벨 `Ideas`, `Research` 표시 확인 (PC/모바일 동일)

### 3) 페이지/템플릿 일관성
- [ ] `Ideas` / `Research` 목록 템플릿 동일 구조 확인
- [ ] `Ideas` / `Research` 상세 템플릿 동일 구조 확인
- [ ] `Research`에만 원문 링크/arXiv 출처 UI 표시 확인
- [ ] Home 최신 콘텐츠 리스트 렌더링 정상
- [ ] About 페이지 Bio/Contact 표시 정상

### 4) 디자인/반응형
- [ ] 라이트/다크 모드 모두 배경+텍스트 대비 WCAG AA 이상
- [ ] 과한 그림자/컬러 카드/조밀한 그리드 없음
- [ ] 모바일/태블릿/데스크톱 레이아웃 깨짐 없음
- [ ] 텍스트 가독성(행간/자간/줄 길이) 문제 없음
- [ ] 다크 모드 전환 시 flash 없음 (새로고침 시 즉시 올바른 테마)
- [ ] 테마 선택값 새로고침 후 유지 (localStorage 퍼시스턴스)
- [ ] 저장값 없을 때 OS prefers-color-scheme 따름
- [ ] prose 본문(코드블록, blockquote, 인라인 코드, 테이블) 다크모드 가독성
- [ ] 원형 테마/언어 버튼이 nav 텍스트 링크와 시각적으로 구분됨

### 5) 검색/공유/분석
- [ ] `sitemap.xml`, `robots.txt` 생성/노출 확인
- [ ] 대표 페이지/포스트 메타(title/description/canonical/OG) 확인
- [ ] ko/en hreflang 쌍 확인
- [ ] JSON-LD(포스트/Person 등) 오류 없음
- [ ] 분석 스크립트가 프로덕션에서만 로드되는지 확인
- [ ] 공통 이벤트 래퍼(`track`) 호출 에러 없음

## B. 콘텐츠 발행 전 체크 (포스트 단위)
### 1) 파일/구조 안전성
- [ ] `posts/<slug>/` 폴더 구조 준수
- [ ] `ko.mdx` 존재
- [ ] `en.mdx` 존재(자동 생성 포함)
- [ ] `cover.webp` 존재(정책상 필수)
- [ ] 본문 이미지 파일 존재 및 불필요 파일 없음
- [ ] 대량 삭제/이동이 포함된 커밋 아님(실수 점검)

### 2) 메타데이터/frontmatter
- [ ] `post_id`가 ko/en 동일
- [ ] `locale` 값 정확 (`ko`, `en`)
- [ ] `title`, `summary`, `published_at`, `updated_at`, `tags`, `cover_image` 존재
- [ ] `published_at`가 기존 글에서 불필요하게 변경되지 않음
- [ ] `cover_image` 경로가 실제 파일과 일치
- [ ] `meta.json` 사용 시(frontmatter와) 핵심 값 충돌 없음

### 3) 이미지/자산 자동정규화
- [ ] 규칙 밖 파일명이 자동 리네임되었는지 확인
- [ ] 리네임 후 `ko.mdx`, `en.mdx` 이미지 경로 치환 완료
- [ ] 이미지 링크 깨짐 없음
- [ ] 커버 이미지 톤이 DESIGN_SYSTEM 규칙과 크게 어긋나지 않음

### 4) 콘텐츠 품질 (ko/en)
- [ ] 한국어 원문 의미/표현이 의도와 일치
- [ ] 영어 번역이 의미 보존(고유명사/링크/코드/수식 변형 없음)
- [ ] 요약이 과장/클릭베이트 없음
- [ ] 태그 수/체계가 과도하지 않음
- [ ] `Research` 글이면 원문(arXiv/출처) 링크 포함
- [ ] 외부 링크(YouTube/arXiv 등) 열림 확인

### 5) 발행 후 반영 확인
- [ ] 목록 페이지에 새 포스트가 노출됨
- [ ] Home 최신 업데이트에 반영됨
- [ ] ko/en 상세 페이지 모두 접근 가능
- [ ] OG 미리보기(대표 1개) 확인
- [ ] 분석 이벤트(카드 클릭/외부링크 클릭) 최소 1회 테스트

## C. Git/릴리스 안전 규칙
- [ ] 콘텐츠 발행 커밋과 사이트 구조 개선 커밋이 분리됨(가능하면)
- [ ] 자동 생성 파일 변경 범위가 예상과 일치
- [ ] 민감정보(API 키/토큰/개인정보) 커밋 없음
- [ ] 푸시 전 `git diff` 최종 확인
- [ ] 이상 시 즉시 롤백 가능한 상태(브랜치/커밋) 확보

## D. v1 범위 확인 (실수 방지)
- [ ] 뉴스레터 구독/발송 기능을 v1에 넣지 않음
- [ ] Substack 연동/자동 발행을 v1에 넣지 않음
- [ ] 과도한 CMS/복잡한 백엔드 기능을 v1에 넣지 않음
