# I18N_ROUTING.md

## 목적
v1(한/영) 사이트의 언어 감지, 라우팅, 전환, fallback 규칙만 정의한다.
(페이지 구조/콘텐츠 모델/SEO 상세는 다른 문서에서 정의)

## 지원 언어 (v1)
- `ko`
- `en`

## 기본 원칙
- 모든 공개 페이지는 언어 prefix를 사용한다.
  - `/ko/...`
  - `/en/...`
- `/` 는 진입점이며, 언어 판별 후 `/ko` 또는 `/en`으로 이동한다.

## 첫 방문 언어 결정 규칙
우선순위:
1. 사용자가 직접 선택해 저장한 언어(쿠키 또는 localStorage)
2. 브라우저 언어(`Accept-Language` / `navigator.language`)
3. 기본값 `en`

판별 규칙:
- 브라우저 언어가 `ko`로 시작하면 `/ko`
- 그 외는 `/en`

## 언어 전환기(Language Switcher) 규칙
- 모든 페이지에 언어 전환기를 노출한다.
- 사용자가 전환 시:
  - 선택 언어를 저장한다.
  - 가능한 경우 같은 콘텐츠의 다른 언어 페이지로 이동한다.
- 저장된 언어는 이후 방문에서 브라우저 언어보다 우선한다.

## 경로 규칙 (라우팅 관점만)
- 홈: `/ko`, `/en`
- 글 목록(What I write): `/ko/write`, `/en/write`
- 읽은 것(What I read): `/ko/read`, `/en/read`
- 소개(About): `/ko/about`, `/en/about`

상세 페이지 패턴:
- 글 상세: `/ko/write/<slug>`, `/en/write/<slug>`
- 읽은 것 상세: `/ko/read/<slug>`, `/en/read/<slug>`

## 번역 페어링 규칙
- 동일 콘텐츠는 한/영에서 같은 `slug`를 사용한다.
- 언어 전환 시 동일 `slug`의 반대 언어 페이지를 찾는다.
- 실제 매핑 정보는 콘텐츠 메타(`meta.json`)를 기준으로 판단한다.

## 번역 누락(fallback) 규칙
### 1) 요청 언어 페이지가 존재함
- 정상 렌더링

### 2) 요청 언어 페이지는 없고 다른 언어만 존재함
- v1 기본 정책: **존재하는 언어 페이지로 리다이렉트**
- 상단 배너로 안내 문구 표시
  - 예: "이 글의 영어 번역은 아직 준비 중입니다."
  - 예: "Korean version is not available yet."

### 3) 두 언어 모두 없음
- 활성 언어 기준 404 페이지 표시 (`/ko/404` 또는 `/en/404`)

## 저장/지속성 규칙
- 명시적 언어 선택이 있을 때만 저장한다.
- 저장값이 있으면 브라우저 언어로 덮어쓰지 않는다.
- 브라우저 언어는 "첫 방문 또는 저장값 없음" 상황에서만 사용한다.

## v1 비범위 (Non-Goals)
- 브라우저 런타임 자동번역
- 지역별 세분 언어(`en-US`, `en-GB` 등)
- 3개 이상 언어 지원

## 수용 기준 (Acceptance Criteria)
- `/` 접속 시 한국어 브라우저는 `/ko`, 그 외는 `/en`으로 이동한다.
- 언어 전환 후 재방문 시 선택 언어가 유지된다.
- 번역이 없는 경우 fallback 동작이 일관되다.
- 404가 활성 언어 기준으로 노출된다.
