# PRD - Terry Taewoong Um Personal Website (v1)

## 1. 문서 목적
- 본 문서는 개인 웹사이트 v1의 제품 목표, 범위, 성공 기준을 정의한다.
- 구현 상세(정보구조, 다국어 라우팅, 콘텐츠 모델, 자동 퍼블리시, 뉴스레터, 디자인 시스템)는 별도 docs 문서에서 정의한다.

## 2. 제품 개요
- AI / Robotics 연구자이자 Beauty AI 스타트업을 창업 및 Exit한 **Terry Taewoong Um**의 개인 브랜드 웹사이트.
- 목적은 다음 4가지를 동시에 달성하는 것:
  - 개인 소개(Bio, Contact, 활동 분야)
  - 직접 작성한 글 아카이브 구축 (+ 구독자 누적)
  - Physical AI 관련 논문/리포트 읽기 및 요약 아카이브 구축 (+ 구독자 누적)

## 3. 핵심 가치 제안
- 한국어 중심 독자에게 친화적인 AI/Robotics/Startup 인사이트 제공
- 영어 사용자도 동일 콘텐츠를 읽을 수 있는 이중언어 경험 제공
- 장기적으로 글/논문요약/영상/외부 플랫폼(Substack 등)으로 확장 가능한 개인 콘텐츠 허브 기반 마련

## 4. 주요 대상 사용자
- 1차 타깃: 한국어 사용자 (AI, 로보틱스, 스타트업, 제조혁신/AI Transformation에 관심 있는 독자)
- 2차 타깃: 영어 사용자 (해외 연구자/실무자/투자자/협력 파트너)
- 운영자(Admin): Terry 본인 (VS Code + GitHub + Claude Code를 통한 콘텐츠 발행)

## 5. v1 제품 범위 (In Scope)
### 5.1 탭 구조 (v1)
- **Home**: 간략한 자기소개 + 최신 업데이트 콘텐츠 리스트
- **Ideas**: 직접 작성한 글(블로그형 리스트)
- **Research**: 읽은 논문/리포트 요약 포스팅(블로그형 리스트)
- **About**: 상세 Bio + Contact + 프로필/소셜 링크

### 5.2 콘텐츠 운영 방식 (v1)
- 운영자는 한국어 원문을 Markdown 기반으로 작성한다.
- Claude Code 기반 퍼블리시 플로우를 통해 한국어/영어 콘텐츠가 동시 발행된다.
- 포스트별 썸네일(cover.webp) 및 메타데이터(meta.json 포함)는 자동 생성 대상이다.
- Home에는 각 탭의 최신 콘텐츠 일부를 요약 카드 형태로 노출한다.

### 5.3 다국어 경험 (v1)
- 브라우저/기기 언어가 한국어면 한국어 사이트를 우선 노출한다.
- 그 외 언어는 영어 사이트를 우선 노출한다.
- 사용자가 수동으로 언어를 변경할 수 있어야 한다.
- 한국어 콘텐츠는 한국어 사이트, 영어 콘텐츠는 영어 사이트에서 대응 노출된다.

## 6. v1 비범위 (Out of Scope)
- Substack 연동/임베드/자동 퍼블리시 (v2 이후 고려)
- 뉴스레터 구독/발송 기능
- 결제/유료 구독/멤버십 기능
- 사용자 로그인/회원 기능
- 댓글 기능
- 고급 검색/추천 시스템
- arXiv 링크만 입력하면 완전 자동 발행되는 파이프라인 (v2 이후 자동화 고도화)
- 복잡한 CMS 또는 웹 WYSIWYG 에디터 내장

## 7. UX/브랜드 방향성 (상위 원칙)
- [swyx](https://swyx.io/) 스타일을 참고하되, Terry의 전문성(AI + Robotics + Startup + Transformation)이 바로 드러나는 랜딩 경험을 지향한다.
- 프로필 사진과 소셜 아이콘 배치를 포함하여 개인 브랜드 허브 느낌을 강화한다.
- 정보량보다 정보구조를 우선하며, 각 탭의 목적이 명확해야 한다.

## 8. 운영/기술 전제 (v1)
- 1차 개발은 **자체사이트만 운영**한다.
- 콘텐츠 원고 작성은 VS Code 기반 Markdown 워크플로우를 사용한다.
- GitHub push 기반 자동 배포를 전제로 한다.
- Claude Code는 번역, 메타데이터 생성, 커버 생성, 게시용 파일 정리를 담당한다.

## 9. 성공 기준 (v1)
### 9.1 제품/구현 성공
- 4개 탭이 동작하며 한/영 페이지 전환이 자연스럽게 이루어진다.
- 한국어 원문 1개 입력 시 한국어/영어 포스트가 모두 발행된다.
- Home에 최신 콘텐츠가 자동 반영된다.
- 포스트마다 썸네일/요약/메타데이터가 일관되게 노출된다.

### 9.2 운영 성공
- 운영자가 복잡한 CMS 없이 Markdown + Claude Code만으로 지속 발행 가능하다.
- 포스트 생성/수정/배포 과정이 반복 가능한 표준 플로우로 정착한다.
- 향후 Substack 확장 시 재사용 가능한 콘텐츠 구조가 확보된다.

## 10. 주요 리스크 및 대응 방향 (요약)
- 번역 품질 일관성: 번역 규칙/검수 포인트를 별도 문서화
- 자동 생성 품질 편차(썸네일/태그/요약): 생성 규칙과 허용 범위 명시
- 다국어 라우팅 혼선: 브라우저 언어 vs 사용자 선택 우선순위 명확화
- 초기 과설계 위험: v1은 단순한 정적 콘텐츠 허브에 집중
- 개발 시 임의의 마크다운 포스팅으로 자체테스트 검증 필요 

## 11. v2 이후 확장 방향 (참고)
- Substack 연동 또는 이전(백필 포함)
- Research 자동화 고도화(arXiv 링크 입력 기반 포스팅 자동 생성)
- 영상/출연 아카이브 확장 (필요 시 별도 탭 추가)
- 뉴스레터 운영 도입 및 구독 세분화
